# Критерии оценки инструкций

## 1. Структура инструкции

### 1.1 Наличие предусловий
Описаны ли условия, которые необходимо выполнить перед началом инструкции.

### 1.2 Наличие цели
Понятно ли, какой результат достигается после выполнения инструкции.

### 1.3 Нумерация шагов
Шаги пронумерованы и каждый шаг — одно действие.

## 2. Полнота содержания

### 2.1 Описание интерфейса
Указаны элементы интерфейса (кнопки, поля, меню), с которыми работает пользователь.

### 2.2 Ожидаемый результат шага
После ключевых шагов описан ожидаемый результат или реакция системы.

### 2.3 Обработка ошибок
Указаны возможные ошибки и способы их устранения.

## 3. Применимость по типу документа

### 3.1 Требования к окружению
Для руководств по развёртыванию: описаны системные требования и требования к окружению.

### 3.2 Права доступа
Для руководств администратора/пользователя: указаны необходимые роли или права.
