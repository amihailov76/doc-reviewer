## Шаг 3. Установите Python 3.11

Для этого проекта требуется **Python 3.11** — не новее. Причина: библиотека PyMuPDF (парсинг PDF) поставляется в виде готовых бинарных сборок только для проверенных версий Python. На Python 3.12+ pip попытается собрать PyMuPDF из исходников, для чего потребуется Visual Studio — и установка завершится ошибкой `Unable to find Visual Studio`.

1. Откройте https://python.org/downloads/release/python-3119/
2. Скачайте **Windows installer (64-bit)** — файл `python-3.11.9-amd64.exe`.
3. При установке **обязательно поставьте галочку «Add Python to PATH»**.
4. Проверьте в командной строке (`Win+R` → `cmd`):

```
py -3.11 --version
```

Ожидаемый результат: `Python 3.11.9`.

> **Если на компьютере уже установлен другой Python** — ничего удалять не нужно. Команда `py -3.11` явно указывает, какую версию использовать. Обе версии будут сосуществовать без конфликтов.

## Шаг 4. Установите Node.js

Если Node.js ещё не установлен:

1. Откройте https://nodejs.org — скачайте версию LTS.
2. Установите с настройками по умолчанию.
3. Проверьте в командной строке:

```
node --version
npm --version
```

---

## Шаг 5. Установите Python-зависимости

Откройте командную строку и перейдите в папку проекта:

```
cd C:\Projects\doc-reviewer
py -3.11 -m pip install -r requirements.txt
```

Установка займёт 1–2 минуты. В конце должно появиться `Successfully installed ...`.

> **Важно:** используйте именно `py -3.11 -m pip`, а не просто `pip`. Это гарантирует, что зависимости установятся для правильной версии Python, даже если на компьютере их несколько.

## Шаг 6. Запустите бэкенд

В той же командной строке (в папке `C:\Projects\doc-reviewer`):

```
py -3.11 run_dev.py
```

Ожидаемый вывод:

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

> **Не закрывайте это окно** — бэкенд должен работать.

---

## Шаг 7. Проверьте бэкенд в браузере

Откройте браузер и перейдите по трём адресам:

**Тест 1** — `http://localhost:8000/api/ping`

Ожидаемый результат:
```json
{"status":"ok","message":"doc-reviewer работает"}
```

**Тест 2** — `http://localhost:8000/api/config/models`

Ожидаемый результат — список из 4 моделей:
```json
{"models":[{"id":"gpt-4o",...},{"id":"gpt-4o-mini",...},...], "active_model":"gpt-4o-mini"}
```

**Тест 3** — `http://localhost:8000/docs`

Откроется интерактивная документация Swagger со всеми эндпойнтами.

Также убедитесь, что создался файл базы данных: `C:\Projects\doc-reviewer\data\db.sqlite`.

## Шаг 8. Установите зависимости фронтенда

Откройте **второе окно** командной строки и выполните:

```
cd C:\Projects\doc-reviewer\frontend
npm install
```

Установка займёт 1–2 минуты, появится папка `node_modules`.

---

## Шаг 9. Запустите фронтенд

В том же втором окне:

```
npm run dev
```

Ожидаемый вывод:

```
  VITE v5.x.x  ready in ...ms
  ➜  Local:   http://localhost:5173/
```

> **Не закрывайте это окно.**

---

## Шаг 10. Проверьте приложение в браузере

Откройте `http://localhost:5173` и проверьте следующее:

- **Боковое меню** слева содержит пять пунктов: Документ, Оценка, Результаты, Снимки, Настройки.
- При переходе на каждый пункт меняется страница — маршрутизация работает.
- На странице **Настройки**:
  - Блок «Статус сервера» показывает зелёный индикатор «Бэкенд работает».
  - Отображается список из 4 моделей из `models.yml` с радиокнопками.
  - По умолчанию выбрана модель `GPT-4o mini`.
  - Кнопка «Сохранить» доступна и при нажатии отправляет запрос на бэкенд без ошибок.

## Устранение проблем

| Проблема | Вероятная причина | Решение |
|---|---|---|
| `python` не найден | Python не в PATH | Переустановите с галочкой «Add to PATH» |
| `pip install` падает с ошибкой `Unable to find Visual Studio` | Установлен Python 3.12+ вместо 3.11 | Установите Python 3.11.9, используйте `py -3.11 -m pip install` |
| `pip install` падает с другой ошибкой | Нет прав | Запустите cmd от имени администратора |
| `http://localhost:8000` не открывается | Бэкенд не запущен | Проверьте первое окно cmd на ошибки |
| Бэкенд падает с `ModuleNotFoundError` | Запущен не из корня проекта | Убедитесь, что вы в `C:\Projects\doc-reviewer`, а не в `backend\` |
| Зелёный индикатор не появился | Фронтенд не видит бэкенд | Убедитесь, что оба окна cmd открыты и работают |
| `npm run dev` — ошибка занятого порта | Порт 5173 уже используется | В `vite.config.js` замените `port: 5173` на `port: 5174` |
