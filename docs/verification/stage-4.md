## О сервисе оценки

### Промпт и критерии

Сервис читает критерии из `criteria.md` при каждом запросе. Системный промпт требует от модели вернуть строго JSON без markdown-блоков. Поддерживаются:
- markdown-блоки ` ```json ``` ` — вырезаются перед парсингом
- `<think>...</think>` блоки reasoning-моделей — вырезаются перед парсингом

### Цветовая схема

| Цвет | Условие |
|---|---|
| 🟢 green | Нет error, не более одного warning |
| 🟡 yellow | Есть warning, нет error или не более одного error |
| 🟠 orange | 2–3 error |
| 🔴 red | 4 и более error |

### Повторные попытки

При ошибке запрос повторяется до 3 раз с паузой 2–6 секунд. После исчерпания попыток оценка останавливается с человекочитаемым сообщением.

### Прокси

Сервис автоматически подхватывает системный прокси через `urllib.request.getproxies()`. Актуально для корпоративных сетей (в том числе при подключении через VPN).

## Выявленные и решённые проблемы

### 1. EventSource требует GET, а не POST

**Ошибка:** `GET /api/evaluation/document/5 → 404 Not Found`

**Причина:** браузерный `EventSource` всегда делает GET-запрос, а эндпойнт был объявлен как `@router.post`.

**Решение:** изменён декоратор на `@router.get`.

### 2. httpx не установлен

**Ошибка:** `ModuleNotFoundError: No module named 'httpx'`

**Решение:** добавлен в `requirements.txt`, установить командой `py -3.11 -m pip install httpx==0.27.0`.

### 3. ModelConfig не является словарём

**Ошибка:** `TypeError: 'ModelConfig' object is not subscriptable`

**Причина:** `get_active_model()` возвращает объект Pydantic, а код обращался к нему через `["id"]`.

**Решение:** заменены обращения `model_cfg["id"]` → `model_cfg.id` и т.п.

### 4. Reasoning-модель возвращает `<think>` блок

**Ошибка:** `Ошибка парсинга JSON: Expecting value: line 1 column 1`

**Причина:** модель `api-llm.ml.ptsecurity.ru` — reasoning-модель, она предваряет ответ блоком `<think>...</think>` с рассуждениями, после которого идёт JSON.

**Решение:** в `_parse_llm_response` добавлено вырезание `<think>` блоков через `re.sub(r"<think>.*?</think>", "", raw_text, flags=re.DOTALL)`.

### 5. Корпоративный прокси блокирует запросы к LLM

**Ошибка:** `httpx.ProxyError: 500 Internal Server Error`

**Причина:** системный прокси `proxy.ptsecurity.com:8080` перехватывал HTTPS-запросы. LLM-сервер `api-llm.ml.ptsecurity.ru` доступен только через корпоративный VPN, при этом запросы должны идти через корпоративный прокси.

**Решение:** `evaluator.py` теперь автоматически подхватывает системный прокси через `urllib.request.getproxies()` и передаёт его в `httpx.Client`.

### 6. Оценка не возобновлялась после обрыва соединения

**Симптом:** оценка останавливалась на середине, перезапуск не помогал — уже оценённые инструкции переоценивались заново.

**Решение:** добавлен параметр `resume=true` в эндпойнт оценки документа и кнопка «↻ Продолжить с места остановки» в интерфейсе. При resume=true инструкции с уже сохранённой оценкой пропускаются.

### 7. Зависание оценки на первых разделах (обрыв SSE без ошибки)

**Симптом:** прогресс останавливается на 2-3 разделе без сообщения об ошибке. При повторном запуске появляется ошибка таймаута.

**Причина:** reasoning-модель долго обрабатывает длинные разделы. Браузер закрывал SSE-соединение не дождавшись ответа, так как сервер молчал всё это время.

**Решение:** heartbeat-пинги через threading + queue. Каждая инструкция оценивается в отдельном потоке, каждые 10 секунд браузеру уходит комментарий `: heartbeat\n\n` — соединение остаётся живым пока модель думает. Если модель не ответила за 180 секунд — инструкция пропускается без остановки всей оценки.

**Результат:** оценка 20-страничного документа заняла 3 минуты без обрывов.

### 8. Поддержка Anthropic API

**Причина:** Anthropic API отличается от OpenAI-совместимого формата:
- Заголовки: `x-api-key` и `anthropic-version: 2023-06-01` вместо `Authorization: Bearer`
- Payload: `system` передаётся отдельным полем, не внутри `messages`
- Ответ: `content[0].text` вместо `choices[0].message.content`

**Решение:** добавлены функции `_build_headers()` и раздельная сборка payload по провайдеру. Парсер ответа поддерживает оба формата.

**Конфигурация в models.yml:**
```yaml
- id: claude-sonnet-4-5
  name: Claude Sonnet 4.5
  provider: anthropic
  base_url: https://api.anthropic.com/v1
  requires_key: true
```

**Переменная окружения:** `ANTHROPIC_API_KEY=sk-ant-...`

### 9. Массовое включение/исключение разделов из оценки

**Запрос:** в больших документах неудобно снимать флажки вручную с каждого раздела.

**Решение:** добавлен эндпойнт `PATCH /api/instructions/document/{id}/include-all` и две кнопки в шапке дерева структуры — **«✓ Включить все»** и **«✗ Исключить все»**. Поддерживается фильтр по классификации через параметр `classification` в теле запроса.

**Изменённые файлы:**
- `backend/routers/instructions.py` — новый эндпойнт `bulk_include`
- `frontend/src/pages/ResultsPage.jsx` — обработчик `handleBulkInclude`, кнопки в шапке дерева
- `frontend/src/styles/ResultsPage.css` — стили шапки дерева
