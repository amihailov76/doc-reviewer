## Ключевые архитектурные решения

### Идентификация инструкций — детерминированный алгоритм

Инструкции определяются по трём признакам без участия LLM:

1. Заголовок содержит отглагольное существительное (девербатив) — через pymorphy3
2. Тело содержит фразу «Чтобы <глагол>...» — регулярное выражение
3. Шаги пронумерованы начиная с «1.» или «1)» — регулярное выражение

Классификация: все три → `instruction`, один-два → `possible`, ноль → `non-instruction`.

LLM используется только для оценки качества инструкций (этап 4), не для их поиска.

### Парсинг PDF — эвристика по размеру шрифта

Заголовки определяются по размеру шрифта относительно медианного по документу. Порог: +15% от медианы. Блоки с известными паттернами (titульные фразы, навигационные блоки, иконки) исключаются из заголовков принудительно.

Подробнее: `development/pdf-parser-troubleshooting.md`

### Цветовая схема оценки

| Цвет | Условие |
|---|---|
| 🟢 Зелёный | Нет ❌, не более одного ⚠️ |
| 🟡 Жёлтый | Есть ⚠️, нет ❌ или не более одного ❌ |
| 🟠 Оранжевый | 2–3 ❌ |
| 🔴 Красный | 4 и более ❌ |

### Снимки (Snapshots)

Первый снимок документа автоматически становится baseline. Последующие снимки сравниваются с baseline по заголовкам разделов. Это позволяет отслеживать деградацию качества документации между версиями.
